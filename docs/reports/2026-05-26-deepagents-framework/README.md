# 任务模式智能体框架 — 能力与执行逻辑

> 配套图：`overview.drawio`（三页）
> 整理日期：2026-05-26

---

## 一句话定义

**任务模式智能体框架** 让大模型像一个员工一样自主完成多步骤的复杂任务：它会自己规划、调用工具、必要时拆活给"专项助手"、敏感动作前等人审批，整个过程的工作进度自动落盘、可中断可恢复。

三大特点：
- **自驱**：大模型自己决定下一步做什么、调用哪个工具
- **可控**：关键步骤可由人审批，结果可追溯、可回滚
- **可持续**：任务状态全程持久化，跨设备 / 跨时间继续

---

## 八大能力要点

### 1. 思考—执行循环
大模型每一轮先"想"再"动"：要么给出最终答复，要么决定调用某个工具拿结果，再回来想下一步。循环直到无需再调工具为止。

### 2. 工具调用（以「知识库检索」为典型）
框架提供一组开箱即用的内置工具，业务也可以挂载自有工具（如知识库检索、报告导出、外部 API 调用）。每个工具有一句话描述，大模型据此决定何时使用。

### 3. 子任务助手
遇到独立、复杂的子问题，主任务可以**把活外包给一个专项助手**：助手有自己独立的上下文和工具集，跑完只把结论回传。主任务的"脑容量"不会被中间细节淹没，也支持嵌套（助手再发起下级助手）。

### 4. 计划清单
大模型可以随时维护一份待办清单（增、删、勾选完成），让多步任务的进度对用户和模型自己都可见。

### 5. 工作区文件操作
框架自带一个"工作区"——大模型可以读、写、编辑、按通配符查找、按正则搜索文件。所有产物（如生成的报告、中间草稿、外部文档拷贝）都落在这里，按需保留。

### 6. 技能包
对常用方法（如"如何写一份调研报告"、"如何审阅一段代码"），可以预先打包成**技能包**放到指定目录。大模型启动时自动看到这些技能的名称和说明，按需采用。

### 7. 记忆库
跨任务沉淀的**长期知识**（项目背景、用户偏好、约定俗成的规范）放在记忆库里。每次任务启动时自动加载到大模型视野中，无需用户每次重复说明。

### 8. 人工审批 + 状态持久化
- **审批**：把哪些工具视作敏感动作可在配置中指定。命中后任务自动暂停，等用户**通过 / 改写参数 / 拒绝**三选一。
- **持久化**：每一步运行时上下文自动写入存储；任务被打断（断网、用户离开、长时任务）后凭"任务线程 ID"可原地继续，工作进度不丢失。

## 典型场景：知识库检索任务

```
用户提问
   ↓
框架装配上下文（系统指令 + 计划 + 技能包 + 记忆库）
   ↓
大模型规划步骤，先调「知识库检索」工具
   ↓
阅读检索结果，判断是否够深入
   ↓
[不够深] 启动一个"研究助手"子任务，让它换关键词再查几轮
   ↓
助手回传结论摘要
   ↓
大模型综合结论，准备写报告
   ↓
要写入工作区文件 → 触发审批 → 用户点"通过"
   ↓
输出结构化结论（要点 + 引用来源）
```

整个过程中，每一步状态自动落盘；如果用户中途关闭浏览器，下次回来还能接着看到答复继续追问。

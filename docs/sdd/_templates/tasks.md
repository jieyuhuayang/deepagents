## 开发模式

- **后端 Test-First**:确定性面按「测试 → 实现」配对编排,测试任务排在配对实现任务之前。基础设施任务(依赖、迁移、配置)无测试配对。
- **前端 Test-Alongside**:组件实现 + vitest 测试在同一任务内完成。
- **E2E 强制**:有 UI 交互类 AC 的 feature,末尾必有一个 Playwright E2E 任务(确定性 fixture 模式)。
- **自包含**:每个任务内联文件、逻辑、验证、AC 上下文,实现时无需回读 spec.md。
